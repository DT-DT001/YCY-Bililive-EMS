# YCY Bililive EMS

面向 Windows 的 B 站直播事件 EMS 控制台。程序使用 Python、Vue 3 和 pywebview 开发，可以监听指定直播间，并根据弹幕、点赞、礼物、上舰等事件控制多台役次元一代或二代双通道设备。

程序使用内置 WebView2 窗口，不依赖外部浏览器。关闭窗口或点击“退出程序”时，会停止所有通道输出并主动断开蓝牙设备。

> 本项目涉及电刺激设备。首次使用必须从低强度开始，并保证“全通道急停”按钮随时可操作。

## 功能

- 使用 `bilibili-api-python` 监听 B 站直播间。
- 支持普通用户、舰长、提督、总督的独立事件配置。
- 支持弹幕、点赞、礼物、进入、关注、分享和上舰事件。
- 支持同时连接多台役次元一代、二代设备。
- 每台设备包含独立的 A、B 通道。
- 每条事件规则可以选择多个设备和通道同步输出。
- 可配置基础强度、基础时间、强度增幅率、时间增幅率、强度上限和时间上限。
- 内置 12 种波形，并支持导入役次元 JSON、郊狼 V2/V3 PULSE 波形。
- 支持单波形循环、多波形顺序播放和随机播放。
- 支持配置 JSON 导入、导出，导出文件不包含设备地址和通道选择。
- 主界面显示通道强度、剩余时间、当前波形、频率、脉冲时间和实时波形曲线。

## 支持的直播事件

可以自动监听：

- 指定弹幕
- 点赞
- 礼物
- 上舰长、上提督、上总督
- 进入直播间
- 关注直播间
- 分享直播间

B 站直播信息流目前不提供单个用户的“离开直播间”和“取关直播间”实时通知。这两类规则保留配置与模拟能力，但默认禁用。

## 首次配置

### 1. 连接设备

1. 打开设备电源和 Windows 蓝牙。
2. 进入“设备管理”页面。
3. 点击“扫描设备”。
4. 为首次发现的设备选择正确的产品代际。
5. 点击“连接”。

两代设备使用相同的 BLE 服务 UUID，协议没有提供查询产品代际的命令，因此首次发现默认选择二代。手动选择后，程序会按照蓝牙地址记忆代际。

### 2. 配置事件

进入“事件配置”，选择一条规则并设置：

- **基础强度**：事件首次触发时的起始强度。
- **基础时间**：事件首次触发时的起始持续时间。
- **强度增幅率**：`事件值 × 强度增幅率`。
- **时间增幅率**：`事件值 × 时间增幅率`。
- **强度上限**：只限制当前事件规则，不影响其他事件。
- **时间上限**：只限制当前事件规则累计的剩余时间。
- **波形**：可以选择一个或多个波形。
- **设备与通道**：同一事件可以同时输出到多个设备的多个通道。

弹幕事件值通常为 `1`；礼物事件值使用电池数量。

示例：

```text
基础强度 = 20
强度增幅率 = 0.1
礼物电池 = 10

首次触发强度 = 20 + 10 × 0.1 = 21
```

```text
基础时间 = 5 秒
时间增幅率 = 0.2
礼物电池 = 10

首次触发时间 = 5 + 10 × 0.2 = 7 秒
同一事件再次触发时只增加 10 × 0.2 = 2 秒
```

### 3. 开始监听

1. 返回“实时控制台”。
2. 输入数字直播间号。
3. 点击“开始监听”。
4. 可以使用顶部的“模拟事件”检查规则和设备输出。

停止监听会清空本次事件记录、清空所有调度状态并停止全部通道输出。

## 调度逻辑

调度以“设备 + 通道”为单位，每个通道完全独立。

### 独立事件状态

每条事件规则在每个目标通道上拥有独立状态，包括：

- 当前累计强度
- 当前剩余时间
- 强度上限
- 时间上限
- 当前波形和播放位置
- 波形播放模式

普通弹幕、舰长弹幕、礼物和上舰等规则不会共享强度、时间或上限。

例如：

1. 普通弹幕事件正在以强度 `100` 输出，上限为 `100`。
2. 舰长弹幕事件触发，计算强度为 `120`，上限为 `120`。
3. 舰长弹幕以 `120` 抢占当前通道。
4. 舰长弹幕结束后，普通弹幕恢复为原来的 `100`。

舰长事件不会被普通弹幕的 `100` 上限限制，普通弹幕也不会继承舰长事件的 `120` 上限。

### 同一事件重复触发

- 强度增加：`当前强度 + 事件值 × 强度增幅率`。
- 时间增加：`当前剩余时间 + 事件值 × 时间增幅率`。
- 重复触发不会再次加入基础时间。
- 强度和时间分别受当前事件自己的上限限制。
- 同一事件在一个通道中始终只有一个状态，不会遗留旧强度任务。

### 不同事件抢占

- 通道始终输出当前强度最高的事件。
- 高强度事件抢占低强度事件。
- 被抢占事件暂停计时，不会在后台消耗剩余时间。
- 高强度事件结束后，恢复剩余事件中强度最高的一个。
- 不同设备、不同通道的调度互不影响。
- 强度相同时，先进入调度器的事件优先。

## 波形播放

内置波形：

`潮汐`、`连击`、`压缩`、`快按`、`渐强`、`心跳`、`节奏`、`呼吸`、`摩擦`、`弹跳`、`波浪`、`敲击`

播放规则：

- 只选择一个波形时，持续循环该波形。
- 顺序模式按照波形选择角标的顺序播放。
- 随机模式会完整播放当前波形后再选择其他波形。
- 随机模式不会连续播放同一个波形。
- 事件结束或被重新激活时，波形从头开始播放。

## 导入波形

### 役次元 JSON

支持 `points`、`data`、`wave` 或 `pulses` 数组。波形点支持以下字段：

```json
{
  "name": "示例波形",
  "points": [
    {"frequency": 20, "pulse_width": 30},
    {"frequency": 30, "pulse_width": 50}
  ]
}
```

频率字段可以使用 `frequency`、`freq`、`hz`；脉冲时间字段可以使用 `pulse_width`、`pulse`、`width`、`us`。

### 郊狼 PULSE

- 支持郊狼 V2 的 `X/Y/Z` 三字节波形。
- 支持郊狼 V3 的四组频率/强度十六进制波形。
- 程序根据郊狼周期公式计算实际频率。
- 根据一个周期中的连续脉冲数量换算 100ms 等效脉冲时间。
- 转换后整条波形统一归一化到役次元 `0-100us` 范围。

役次元实时模式最快每 100ms 更新一次，郊狼 V3 每 100ms 包含四个 25ms 子点，因此程序会将四个子点转换为一个 100ms 等效点。两种设备硬件结构不同，转换结果是尽可能保持频率、节奏和相对脉冲量的近似。

## 配置导入导出

事件配置页面支持导出和导入 JSON。

导出内容包括：

- 事件参数
- 强度与时间上限
- 波形选择
- 波形播放顺序和模式

不会导出：

- 蓝牙设备地址
- 设备产品代际
- A/B 通道目标

这样可以将事件配置迁移到其他电脑，并重新选择当地连接的设备。

## 急停和退出

- “全通道急停”会立即清空调度器并向所有活动通道发送停止输出。
- “停止监听”会停止直播监听、清空事件记录并停止输出。
- 点击侧栏“退出程序”或关闭窗口，会停止输出并断开蓝牙。
- 正常情况下，蓝牙连接会在 10 秒内释放。

## 常见问题

### 点击开始监听后连接失败

- 确认直播间号为纯数字。
- 检查 Clash、代理软件或系统代理是否影响 B 站 WebSocket。
- 检查 Windows 防火墙是否允许程序联网。
- B 站接口发生临时变化时，可以稍后重试。

### 扫描不到设备

- 确认设备已开机且没有连接到其他手机或电脑。
- 确认 Windows 蓝牙已开启。
- 将设备靠近电脑后重新扫描。
- 部分蓝牙适配器需要更新驱动。

### 设备连接但没有输出

- 检查事件是否选择了对应设备和 A/B 通道。
- 检查电极片连接状态。
- 使用“模拟事件”检查配置。
- 从较低强度开始逐步增加。

### 程序图标没有立即更新

Windows 可能缓存旧 EXE 图标。将旧版本删除、解压到新目录，或重启资源管理器后即可刷新。

## 安全提示

- 首次使用必须从低强度开始。
- 不要将电极放置在头部、颈前、胸部两侧或让电流穿过心脏的位置。
- 佩戴心脏起搏器或其他植入式电子设备者不应使用。
- 身体不适、皮肤受损、饮酒或意识不清时不要使用。
- 不要依赖软件作为唯一安全措施，确保可以直接关闭设备电源。
- 本项目仅提供软件控制工具，使用者需自行承担设备使用风险。
