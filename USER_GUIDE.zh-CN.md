# YCY Bililive EMS 详细使用说明

适用版本：`v1.1.1`

本说明面向直接下载 Windows 免安装包的用户。建议首次使用时按照“第一次使用完整流程”逐步操作，不要跳过低强度测试和急停测试。

> 安全警告：本软件会控制电刺激设备。首次使用必须从很低的强度开始，确保电极位置安全，并保证设备电源和软件的“全通道急停”随时可以操作。

## 1. 软件用途

YCY Bililive EMS 用于监听指定 B 站直播间，将直播事件转换为役次元一代或二代电击器的输出。

主要能力：

- 监听指定弹幕、点赞、礼物、上舰、进入、关注和分享事件。
- 按普通用户、舰长、提督、总督分别配置。
- 同时连接多台一代、二代设备。
- 每台设备的 A、B 通道可以独立配置。
- 一个事件可以同时触发多个设备、多个通道。
- 为每条事件独立设置强度、持续时间、增幅、上限和波形。
- 支持内置固定波形、役次元 JSON 和郊狼 PULSE 波形。
- 支持事件抢占、排队和恢复。

## 2. 压缩包内容和启动方法

完整解压 ZIP 后，目录中主要包含：

- `YCY-Bililive-EMS.exe`：程序入口。
- `_internal`：程序运行依赖，不能删除。
- `README.md`：项目和开发说明。
- `使用说明.md`：本用户手册。
- `CHANGELOG.md`：版本更新记录。

启动步骤：

1. 将 ZIP 完整解压到普通文件夹。
2. 不要直接在压缩包预览窗口中运行。
3. 双击 `YCY-Bililive-EMS.exe`。
4. 等待内置窗口打开。

程序默认使用内置 WebView2 窗口，不需要外部浏览器。关闭窗口或点击“退出程序”时，程序会停止输出并断开蓝牙。

如果当前电脑的 .NET Framework、WebView2 或内置窗口组件不兼容，程序会自动打开默认浏览器进入兼容模式，不会再因 `Python.Runtime.dll` 加载失败而直接退出。兼容模式的监听和设备控制功能与内置窗口一致；使用结束时必须点击页面中的“退出程序”，不要只关闭浏览器标签页。

如果双击后没有窗口：

1. 等待 Windows Defender 完成首次检查。
2. 确认整个目录已经解压。
3. 确认 `_internal` 文件夹与 EXE 在同一目录。
4. 安装或修复 Microsoft Edge WebView2 Runtime。
5. 如果仍无法打开内置窗口，可等待程序自动进入浏览器兼容模式；也可设置环境变量 `YCY_BROWSER_MODE=1` 后启动程序。
5. 检查任务管理器中是否已有旧版本进程。

## 3. 界面区域说明

左侧菜单包含四个主要页面。

### 3.1 实时控制台

用于开始或停止直播监听，并观察当前运行状态。

- **直播间号**：输入纯数字 B 站直播间号。
- **开始监听**：连接直播间并开始接收事件。
- **停止监听**：停止监听、清空本次事件记录、清空事件队列并停止全部输出。
- **已连接设备**：当前蓝牙连接成功的设备数量。
- **本次事件**：本次监听期间收到的事件数量。
- **活动通道**：当前正在输出的 A/B 通道数量。
- **设备卡片**：显示设备代际、在线状态、通道强度、波形、总剩余时间和输出参数。
- **实时事件**：显示最近收到的用户名、事件类型、内容和事件值。

固定波形播放时，界面以较大文字显示波形名称和强度。导入的自定义波形播放时，界面显示频率与脉冲时间曲线。

### 3.2 事件配置

用于配置每一种直播事件触发后的输出行为。

左侧为事件列表，右侧为所选事件的详细参数。

顶部按钮：

- **模拟事件**：模拟当前选中的已启用规则，测试值固定为 `10`。
- **导出配置**：将事件参数保存为 JSON。
- **导入配置**：从 JSON 恢复事件参数。
- **保存配置**：将当前修改写入程序配置。

重要：修改规则后必须点击“保存配置”。模拟事件会自动先保存，但真实直播事件不会替你保存尚未提交的界面修改。

### 3.3 设备管理

用于扫描、设置代际、连接和断开设备。

- **扫描设备**：搜索使用 FF30 服务 UUID 的设备。
- **产品代际**：选择一代或二代。
- **电量**：显示设备报告的剩余电量。
- **连接/断开**：建立或关闭蓝牙连接。
- **设备地址**：设备的蓝牙地址，用于区分多台设备。

一代和二代使用相同的服务 UUID，协议不能自动查询代际。首次发现默认可能为二代，必须根据实际设备手动选择。程序会记住这个蓝牙地址对应的代际。

### 3.4 波形库

用于查看内置波形和导入自定义波形。

- 内置 12 个波形：潮汐、连击、压缩、快按、渐强、心跳、节奏、呼吸、摩擦、弹跳、波浪、敲击。
- 支持役次元 JSON。
- 支持郊狼 V2/V3 PULSE。
- 支持 `Dungeonlab+pulse` 多小节格式。
- 支持 `.pulse`、`.pules`、`.json` 和 `.txt` 文件。

## 4. 第一次使用完整流程

### 第一步：准备设备

1. 打开 Windows 蓝牙。
2. 打开役次元设备电源。
3. 关闭可能占用设备的手机 APP。
4. 不要让设备同时连接其他电脑或手机。
5. 首次测试先不要佩戴在高风险部位，并将强度设得很低。

### 第二步：扫描并连接

1. 打开“设备管理”。
2. 点击“扫描设备”。
3. 找到设备后选择正确的“一代”或“二代”。
4. 点击“连接”。
5. 等待状态显示在线和电量。

连接多台设备时，对每台设备重复以上步骤。

### 第三步：配置一条低强度测试规则

建议先使用“指定弹幕 · 普通用户”：

1. 打开“事件配置”。
2. 选择“指定弹幕 · 普通用户”。
3. 打开启用开关。
4. 触发弹幕填写容易测试的文字，例如 `测试电击`。
5. 基础强度设为较低值，例如 `5`。
6. 基础时间设为 `1` 秒。
7. 强度增幅率设为 `0`。
8. 时间增幅率设为 `0`。
9. 强度上限设为 `5`。
10. 时间上限设为 `1` 秒。
11. 只选择一个内置波形，例如“潮汐”。
12. 在“输出设备与通道”中只勾选一个测试通道。
13. 点击“保存配置”。

### 第四步：测试急停

1. 点击顶部“模拟事件”。
2. 确认对应设备和通道短暂输出。
3. 在输出过程中点击左侧“全通道急停”。
4. 确认强度立即归零、剩余时间清零。

确认急停有效后再继续。

### 第五步：开始监听

1. 返回“实时控制台”。
2. 输入纯数字直播间号。
3. 点击“开始监听”。
4. 等待状态显示“房间 XXXXX”。
5. 在直播间发送配置的测试弹幕。
6. 观察实时事件和设备通道状态。

## 5. 支持的事件及事件值

每次事件都会携带一个“事件值”。事件值用于计算首次强度、首次时间和重复触发增幅。

| 事件 | 自动监听 | 通常使用的事件值 |
| --- | --- | --- |
| 指定弹幕 | 支持 | `1` |
| 点赞 | 支持 | B 站消息中的点赞数量 |
| 礼物 | 支持 | 礼物总电池数 |
| 上舰长 | 支持 | 购买数量 |
| 上提督 | 支持 | 购买数量 |
| 上总督 | 支持 | 购买数量 |
| 进入直播间 | 支持 | `1` |
| 关注直播间 | 支持 | `1` |
| 分享直播间 | 支持 | `1` |
| 离开直播间 | 不支持自动推送 | 仅可配置和模拟 |
| 取关直播间 | 不支持自动推送 | 仅可配置和模拟 |

弹幕、点赞、礼物、进入、关注和分享可按以下身份分别配置：

- 普通用户
- 舰长
- 提督
- 总督

“上舰长”“上提督”“上总督”本身是独立事件，不需要再选择身份。

## 6. 每项配置的用途

### 6.1 规则名称

仅用于界面显示和事件记录，可以改成容易理解的名称，不影响匹配逻辑。

### 6.2 启用

关闭后，该规则不会被真实事件或默认模拟选择触发。

“离开”和“取关”因 B 站没有可靠的单用户实时推送，界面会提示“暂无推送”，不能自动启用。

### 6.3 触发弹幕

仅指定弹幕规则显示。

- 填写文字：弹幕内容包含该文字时触发，匹配区分大小写。
- 留空：该身份的任意弹幕均可触发。
- 可使用“添加指定弹幕”建立多条不同关键词规则。

新增的指定弹幕规则可单独删除，内置规则不可删除。

### 6.4 用户身份

用于匹配发送事件的用户身份。

例如：

- 普通用户发送礼物，只匹配“礼物 · 普通用户”。
- 舰长发送礼物，只匹配“礼物 · 舰长”。

不同身份规则互相独立，可以设置不同强度、时间、上限、波形和通道。

### 6.5 基础强度

事件第一次进入该通道时的基础强度，范围为 `0-276`。

首次计算：

```text
首次强度 = 基础强度 + 事件值 × 强度增幅率
```

计算结果不能超过该规则自己的强度上限。

自定义波形播放期间，事件强度保持不变；波形只改变频率和脉冲时间。

### 6.6 基础时间

事件第一次进入该通道时提供的基础持续时间，单位为秒。

首次计算：

```text
首次时间 = 基础时间 + 事件值 × 时间增幅率
```

计算结果不能超过该规则自己的时间上限。

### 6.7 强度增幅率

决定事件值对强度的影响。

示例：

```text
基础强度 = 20
强度增幅率 = 0.1
礼物 = 10 电池

首次强度 = 20 + 10 × 0.1 = 21
```

同一事件重复触发时：

```text
新强度 = 当前强度 + 本次事件值 × 强度增幅率
```

重复触发不会再次添加基础强度。

### 6.8 时间增幅率

决定事件值对持续时间的影响。

示例：

```text
基础时间 = 5 秒
时间增幅率 = 0.2
礼物 = 10 电池

首次时间 = 5 + 10 × 0.2 = 7 秒
```

同一事件再次触发：

```text
增加时间 = 本次事件值 × 时间增幅率
```

此例只增加 `2` 秒，不会再次增加 `5` 秒基础时间。

### 6.9 强度上限

只限制当前规则。

例如普通弹幕上限为 `100`、舰长弹幕上限为 `120`：

- 普通弹幕最多到 `100`。
- 舰长弹幕最多到 `120`。
- 舰长弹幕不会被普通弹幕的 `100` 限制。
- 舰长事件结束后，恢复普通弹幕原来的状态。

### 6.10 时间上限

只限制当前规则在一个通道中的累计剩余时间。

即使连续收到大量事件，该规则的剩余时间也不会超过时间上限。

### 6.11 波形选择

复选框的勾选顺序会显示为角标 `1、2、3...`，这个顺序就是“顺序播放”的播放顺序。

- 只选择一个波形：持续循环该波形，不显示模式选择。
- 选择多个波形：可以选择顺序或随机。

### 6.12 播放模式

- **单波形播放**：只有一个波形时自动循环。
- **顺序**：按选择角标顺序完整播放，每个波形播完后切换下一个。
- **随机**：当前波形完整播完后，从其他波形中随机选择；不会连续重复同一个波形。

事件结束或重新激活后，波形从头开始。

### 6.13 输出设备与通道

指定该事件输出到哪些设备和通道。

- 同一规则可以勾选多个设备。
- 每台设备可以选择 A、B 或两个通道。
- 每个目标通道拥有独立队列。
- 不同通道互不影响。

如果没有勾选任何目标，事件会被记录，但不会产生设备输出。

## 7. 强度、时间和抢占逻辑

调度单位是“设备 + 通道”。例如：

- 设备 1 的 A 通道是一条独立队列。
- 设备 1 的 B 通道是另一条独立队列。
- 设备 2 的 A/B 又是另外两条队列。

### 7.1 同一事件重复触发

同一规则在同一通道中只保留一个状态。

- 强度按本次事件值继续增加。
- 时间只增加本次增幅时间。
- 不重复添加基础时间。
- 强度和时间分别受该规则自己的上限限制。

### 7.2 不同事件同时存在

通道始终输出强度最高的事件。

- 高强度事件会抢占低强度事件。
- 被抢占事件暂停计时。
- 被抢占事件不会在后台消耗剩余时间。
- 高强度事件结束后，恢复剩余事件中强度最高的一个。
- 强度相同时，先进入队列的事件优先。

示例：

1. 普通弹幕以强度 `20`、剩余 `10` 秒运行。
2. 礼物事件以强度 `40`、持续 `5` 秒触发。
3. 礼物抢占普通弹幕。
4. 普通弹幕暂停，剩余时间不会继续减少。
5. 礼物结束后，普通弹幕从剩余时间继续运行。

### 7.3 总剩余时间

主界面的“剩余时间”显示该通道队列中所有事件的总剩余时间，不只是当前正在输出的事件。

## 8. 波形和设备协议

### 8.1 内置 12 个波形

内置波形使用设备协议的固定模式 `1-12`，不是由软件逐点发送自定义曲线。

主界面的固定波形图或参考信息不代表设备实时回传的电流采样。协议不会返回真实频率、脉冲时间或电流波形。

### 8.2 自定义波形

导入波形使用实时自定义模式。

每个波形点包含：

- `frequency`：频率，范围 `1-100Hz`。
- `pulse_width`：脉冲时间，范围 `0-100us`。

事件配置中的强度保持恒定，波形只动态改变频率和脉冲时间。

### 8.3 一代设备

- A、B 参数完全相同时，程序发送一条通道号 `0x03` 的同步报文。
- A、B 参数不同时，分别发送单通道报文。
- 任意两条控制报文至少间隔约 `120ms`。
- 等待发送期间只保留每个通道最新波形点，避免旧数据堆积。

### 8.4 二代设备

二代固定模式和实时模式是整台设备共用的报文类型。

当 A、B 同时输出，一边使用内置固定波形、另一边使用导入波形时，程序会使用实时模式维持两个通道输出。

## 9. 导入波形

### 9.1 役次元 JSON

示例：

```json
{
  "name": "示例波形",
  "points": [
    {"frequency": 20, "pulse_width": 30},
    {"frequency": 30, "pulse_width": 50}
  ]
}
```

数组名称支持：

- `points`
- `data`
- `wave`
- `pulses`

频率字段支持：

- `frequency`
- `freq`
- `hz`

脉冲时间字段支持：

- `pulse_width`
- `pulse`
- `width`
- `us`

### 9.2 郊狼 PULSE

支持：

- 郊狼 V2 三字节波形。
- 郊狼 V3 波形。
- Dungeonlab 多小节长波形。

转换原则：

- 郊狼频率转换为役次元频率。
- 郊狼的波形强度维度转换为役次元脉冲时间。
- 事件配置中的电击强度不会随波形点改变。
- 郊狼 V3 的多个短子点会转换为役次元约 100ms 的等效点。

两种设备硬件和参数模型不同，因此转换结果是近似体感，不保证与郊狼设备完全一致。

旧版本已经导入的波形可能保存了旧转换结果。升级后建议用原始 PULSE 文件重新导入。

## 10. 配置导入和导出

### 10.1 导出配置

在“事件配置”页面点击“导出配置”，选择保存位置。

导出内容：

- 规则名称和启用状态
- 触发弹幕和用户身份
- 强度、时间、增幅率和上限
- 波形选择
- 波形选择顺序
- 播放模式

不会导出：

- 蓝牙地址
- 产品代际
- 设备和 A/B 通道选择
- 直播间号

这样可避免把旧电脑的设备绑定带到另一台电脑。

### 10.2 导入配置

1. 先扫描并连接当前电脑上的设备。
2. 打开“事件配置”。
3. 点击“导入配置”。
4. 选择之前导出的 JSON。
5. 程序会导入事件参数，同时保留当前已有的设备和通道选择。
6. 检查每条规则的输出目标。
7. 点击“保存配置”。

如果当前规则 ID 与配置文件不一致，新规则可能没有目标通道，需要手动选择。

## 11. 模拟事件

“模拟事件”用于不连接直播间时测试规则和设备。

- 在事件配置页面选中一条已启用规则后点击模拟，会模拟该规则。
- 如果当前规则未启用，程序会选择第一条已启用规则。
- 模拟事件值固定为 `10`。
- 模拟前程序会自动保存当前界面配置。

测试礼物规则时，模拟值 `10` 相当于用事件值 `10` 计算，不代表真实支付。

## 12. 停止、急停和退出的区别

### 全通道急停

- 立即清空所有通道的事件队列。
- 停止所有设备输出。
- 不主动停止直播间监听。
- 后续新的直播事件仍可能再次触发输出。

如果希望急停后不再收到新触发，还要点击“停止监听”。

### 停止监听

- 停止 B 站连接。
- 清空本次事件记录。
- 清空全部通道队列。
- 停止全部输出。

### 退出程序

- 停止输出。
- 停止监听。
- 保存配置。
- 主动断开蓝牙。
- 正常情况下蓝牙会在约 10 秒内释放。

## 13. 推荐配置方法

### 普通弹幕

- 基础强度较低。
- 基础时间较短。
- 增幅率设为 `0` 或很小。
- 使用柔和波形。

### 点赞

B 站点赞事件值可能一次包含多个点赞，因此先使用很小的增幅率测试，避免一次消息产生过大增幅。

### 礼物

- 根据电池数设置增幅率。
- 设置明确的强度上限和时间上限。
- 先用模拟值 `10` 检查计算。
- 再用小礼物进行真实测试。

### 上舰

上舰事件通常较少，可使用独立波形和稍长时间，但仍应设置安全上限。

### 多设备直播展示

- 每台设备的 A/B 通道分别配置。
- 先逐个通道测试，再启用多个通道。
- 主界面最多优先展示前四台设备。
- 避免在未测试的情况下同时开启所有高强度规则。

## 14. 常见问题

### 14.1 扫描不到设备

- 确认 Windows 蓝牙已开启。
- 确认设备已开机。
- 关闭手机上的官方 APP。
- 确认设备没有连接其他程序。
- 将设备靠近电脑。
- 更新蓝牙适配器驱动。
- 断电重启设备后重新扫描。

### 14.2 显示 Device was not found

- 设备可能已经停止广播。
- 重新打开设备电源。
- 点击“扫描设备”刷新发现结果。
- 确认没有其他程序占用设备。

### 14.3 设备已连接但不输出

- 检查规则是否启用。
- 检查是否点击了“保存配置”。
- 检查规则是否选择了当前设备和通道。
- 检查基础强度、基础时间是否大于 `0`。
- 检查电极连接状态。
- 使用“模拟事件”测试。

### 14.4 直播间无法连接

- 直播间号必须为纯数字。
- 检查网络连接。
- 检查 Windows 防火墙。
- 检查 Clash、代理或系统代理是否影响 B 站 WebSocket。
- 点击重试。
- 心跳超时后程序会自动销毁旧连接并退避重连。

### 14.5 一代设备错误码 4

错误码 4 表示设备拒绝上一条控制数据，不等于设备损坏。

- 确认产品代际选择为“一代”。
- 使用本版本或更新版本。
- 确认导入波形参数在有效范围内。
- 停止输出并断开设备。
- 重启设备后重新扫描、连接。
- 记录界面显示的最近通道、强度、频率和脉冲时间。

### 14.6 关闭程序后蓝牙仍显示连接

- 正常退出后等待最多约 10 秒。
- 不要使用任务管理器强制结束，强制结束可能使 Windows 延迟释放 BLE。
- 如果长时间不释放，关闭设备电源后重新打开。

### 14.7 配置导入后没有输出

导出文件故意不包含设备和通道绑定。导入后需要检查当前电脑上的输出目标，并点击“保存配置”。

### 14.8 波形显示和实际体感不完全一致

- 内置固定模式由设备固件生成，软件无法读取真实输出波形。
- 自定义波形图显示的是软件发送的频率和脉冲时间参数。
- 图形不是示波器测得的真实电流。
- 电极接触、皮肤状态、设备硬件和输出负载都会影响体感。

## 15. 安全注意事项

- 首次使用必须从低强度开始。
- 每条规则都应设置合理的强度上限和时间上限。
- 不要将电极放在头部、颈前、胸部两侧。
- 不要让电流路径穿过心脏。
- 不要在破损、发炎或感觉异常的皮肤上使用。
- 佩戴心脏起搏器或其他植入式电子设备者不应使用。
- 孕期、严重心血管疾病或癫痫患者不应使用。
- 饮酒、服用影响意识的药物、困倦或意识不清时不要使用。
- 不要在驾驶、操作机械、洗澡或水边使用。
- 不要把软件当作唯一安全措施，确保可以直接关闭设备电源。
- 直播前先使用模拟事件逐条测试所有会启用的规则。
- 修改强度、增幅率、上限、波形或通道后，应重新进行低强度测试。
- 任何不适应立即急停、关闭设备并移除电极。

## 16. 建议的直播前检查清单

1. 设备电量充足。
2. 产品代际选择正确。
3. 所有需要的设备已连接。
4. 电极位置安全、接触正常。
5. 每条启用规则已检查输出目标。
6. 强度上限和时间上限合理。
7. 礼物和点赞增幅率已用模拟值测试。
8. 全通道急停测试有效。
9. 直播间号正确。
10. 开始监听后状态显示已连接。
11. 软件窗口和设备电源都能随时操作。

完成以上检查后再正式使用。
